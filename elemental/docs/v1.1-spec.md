# Muju Hono Tanka v1.1 Spec

## Overview

This spec captures balance changes, rule modifications, UI improvements, and bug fixes based on playtesting feedback.

---

## 1. Rule Changes

### 1.1 Action Economy
- **Actions per turn:** 6 (up from 4)
- **Repeated actions:** No limit on a unit taking the same action multiple times per turn (e.g., can move twice if speed allows)

### 1.2 Summoning Sickness Removed
- Units can act immediately when placed (no summoning sickness)
- Units can act immediately after promotion
- *Note: If summoning sickness were kept, it should expire at end of the turn the unit was summoned (e.g., AI-summoned units can act on AI's next action, not next full turn)*

### 1.3 Promotion Cost Formula
**Current:** Cost = next tier number (T1→T2 costs 2, T2→T3 costs 3, etc.)
**New:** Cost = difference in unit costs

```
Promotion Cost = TargetTierCost - CurrentTierCost

Examples (Fire):
- Fire_1 (cost 1) → Fire_2 (cost 3): costs 2
- Fire_2 (cost 3) → Fire_3 (cost 6): costs 3
- Fire_3 (cost 6) → Fire_4 (cost 10): costs 4
```

### 1.4 Build Queue Persistence
- Units in queue should **never be auto-deleted**
- If a unit is ready (turnsRemaining = 0) but can't spawn, it stays in queue indefinitely
- If tech is lost (e.g., anchor destroyed), queued units remain and can spawn when tech is restored
- Decrementing turnsRemaining below 0 should not delete the unit

### 1.5 Phase Auto-Transitions
- **Placement phase:** Auto-end when no units in build queue AND no resources to spend
- **Queue phase:** Auto-end when out of resources (can't queue or promote anything)

## 2. Element System Overhaul

### 2.1 Current Problems
- Two separate RPS triangles feel redundant
- Rock-paper-scissors feels watered down
- 6 elements without clear identity

### 2.2 New Archetype Triangle
```
Rush > Balanced > Expand > Rush

Rush beats Balanced (aggressive tempo overwhelms)
Balanced beats Expand (can pressure before economy kicks in)
Expand beats Rush (survives initial aggression, out-economies)
```

### 2.3 Element Archetype Assignments
| Archetype | Elements | Identity |
|-----------|----------|----------|
| Rush | Fire, Lightning | High ATK or SPD, low DEF, Mining 0-1 |
| Balanced | Water, Wind | Moderate all stats, Mining 1-3 |
| Expand | Plant, Metal | High DEF + Mining, low ATK + SPD |

### 2.4 Stat Identity per Element
Each element should be clearly **Strong / Medium / Weak** on each stat:

| Element | ATK | DEF | SPD | MINE | Notes |
|---------|-----|-----|-----|------|-------|
| Fire | Strong | Weak | Med | Weak | Aggressive fighter |
| Lightning | Med | Weak | Strong | None→Weak | Speed specialist, can't mine early |
| Water | Med | Med-Strong | Weak | Med | Defensive all-rounder |
| Wind | Med-Strong | Med | Med-Strong | Weak→Med | Mobile striker |
| Plant | Weak | Med | Weak | Strong | Economic specialist |
| Metal | Weak | Strong | Weak | Med-Strong | Tank/defender |

### 2.5 Elemental Combat Modifier
**Design Question:** Should we keep the +1/-1 ATK modifier or simplify?

Options:
1. **Keep two triangles** (Fire→Plant→Water→Fire, Lightning→Metal→Wind→Lightning)
2. **Single archetype triangle** (Rush elements get +1 vs Balanced, Balanced +1 vs Expand, Expand +1 vs Rush)
3. **Remove elemental modifiers** (rely purely on stat differences)

*Recommendation: TBD based on further playtesting*

## 4. UI Changes

### 4.1 Visual Polish
- **Unit borders:** Make black/white player borders more obvious/thicker
- **Enemy unit tap:** Tapping enemy unit should show their stats (same UnitInfo panel)

### 4.2 Layout Restructure
**Current:** Opponent info at top, board in middle, player info at bottom

**New Layout (top to bottom):**
1. Player actions remaining
2. Player build queue
3. Unit info card (when unit selected)
4. Game board
5. Opponent info (resources, build queue)

**Key constraint:** Board should not move when unit card opens. If it would, unit card should be a popover/overlay instead of pushing content.

### 4.3 Undo System
- Add undo button that reverts actions
- Can undo back to start of player's turn
- Maintains a queue of actions taken this turn
- Each undo pops the most recent action and restores previous state

### 4.4 AI Turn Recap
- After AI turn completes, show a summary of:
  - Units placed (which units, where)
  - Promotions made (which unit promoted to what)
  - Actions taken (moves, attacks, mines)
- Could be a dismissable overlay or log panel

## 5. AI Improvements

### 5.1 Resignation Logic
AI should recognize when it's losing and resign to avoid prolonged hopeless games.

Possible resignation triggers:
- No units remaining and can't spawn any
- Massive resource/unit disadvantage with no recovery path
- All spawn zones permanently blocked

### 5.2 Counter Rush Strategy
**Current problem:** AI gets roflstomped by mass Fire_1 rush.

AI needs to:
- Recognize when being rushed
- Prioritize defensive unit production (Metal/Water)
- Consider positioning to protect spawn zones
- Value early aggression against pure-economy openings

## 7. Open Questions

### 7.1 Element Identity Brainstorm
Need to ensure Rush > Balanced > Expand is intuitive for all element pairs:

| Attacker | Defender | Expected Advantage |
|----------|----------|-------------------|
| Fire | Water | Fire (Rush > Bal) |
| Fire | Wind | Fire (Rush > Bal) |
| Fire | Plant | Fire (Rush > Exp)? — currently Rush loses to Expand |
| Fire | Metal | Fire (Rush > Exp)? — currently Rush loses to Expand |
| Lightning | Water | Lightning (Rush > Bal) |
| Lightning | Wind | Lightning (Rush > Bal) |
| ... | ... | ... |

**Issue:** Rush > Expand contradicts the archetype triangle (Expand should beat Rush).

**Options:**
1. Make element modifiers independent of archetype matchups
2. Lean into archetype triangle and drop per-element RPS
3. Have elements matter for some matchups, archetypes for others

### 7.2 Starting Units
With new stats, what should starting units be?
- Current: Fire_1, Water_1, Plant_1 (one from each archetype)
- Alternative: Let player choose? Different starts for different strategies?
