# TESSER

A dueling tactics game where every piece is conserved hypervolume worn as geometry: fold a tower into a spear, fold a spear into a slab ‚Äî never gain or lose an ounce. Combat is projection; damage is overlap. Winner of the July 2026 design tournament (`docs/game-design-tournament-2026-07.md` at the repo root); full ruleset and stage plan in `SPEC.md`.

The first real game built on the platform after the pebble-duel worked example ‚Äî five of the six kits are load-bearing (`@deev/llm` is deliberately deferred to the post-V1 shape-drop pipeline; see SPEC ¬ß3).

```bash
cd platform
pnpm install
pnpm --filter @deev/example-tesser typecheck   # src + web projects
pnpm --filter @deev/example-tesser test        # 68 tests (~2.5 min; the lab series dominates)
pnpm --filter @deev/example-tesser dev         # vite dev server ‚Äî play in the browser
```

## Layout

- `SPEC.md` ‚Äî the exact V1 ruleset (board, pieces, classes/speeds, slide-then-strike combat, folds, endings) and the four-stage build plan. The single source of truth; `version` bumps on any logic change.
- `src/game.ts` ‚Äî the `GameDef` (`@deev/core`): pure, deterministic, rng-free `apply`; attacks and folds are commit points.
- `src/persist.ts` ‚Äî versioned save envelope (zod-validated) shared by the web UI's localStorage store.
- `src/eval.ts` / `src/bots.ts` ‚Äî `composeEval` factors (measure-diff, threat, mobility, advance) and `tesserMinimaxBot` (`@deev/ai` alpha-beta, move ordering, transposition table).
- `data/campaign.csv` + `src/puzzles.ts` ‚Äî 12 taught missions in the self-describing CSV format, schema-validated (`@deev/content`) and machine-verified: every mission is provably winnable by elimination for the player (minimax on both seats, seeds 1/2/3) and not pre-solved. `verifierIssues(...) === []` is a CI test.
- `web/` ‚Äî portrait mobile UI on the `@deev/ui` shell: tap-select-then-confirm actions, fold shape‚Üíanchor picker, three AI difficulty budgets, save/resume, commit-point-aware undo.
- `REPORT.md` ‚Äî deterministic lab report (`@deev/lab`), regenerated by the test suite and byte-identical across runs: sensitivity gate, strength ladder, conservation invariant, first-mover measurement.

## Known V1 balance notes (from the lab and the campaign verifier)

- Equal-budget minimax mirrors stall to 100-ply adjudicated draws ‚Äî the deterministic engine means that's one game replayed, not sixty samples. The SPEC ¬ß4 balance levers (speeds, reach, plyCap, armies) are the tuning surface.
- Small pieces can fold into fast lines/points and evade indefinitely on open boards; campaign missions are authored around the depth-3 horizon accordingly. A future anti-stall lever (e.g. adjudication favoring the aggressor, or shrinking boards) is recorded, not built.
- `@deev/ai` node-cap caveat: when `maxNodes` trips mid-root, unscored root actions are dropped, which can leave a fold-only action list under attack-first ordering. The campaign verifier therefore uses uncapped depth-3; the interactive bot keeps the cap (acceptable in play, documented in `src/puzzles.ts`).
